# 云服务器实测指南

> 日期：2026-07-03。Firecracker 需要 `/dev/kvm`：**裸金属或支持嵌套虚拟化的云 VM**。本指南基于逐厂商核实（2026-07 一手来源），给出在云服务器上开发、测试、跑基准的完整路径。
> 两个重要修正：① "Hetzner Cloud VM 暴露 vmx" 的社区传言**不成立**（官方 FAQ 明确不支持，Alex Ellis 2025 实测 CCX 也无 /dev/kvm）；② "AWS 只有 .metal" **已过时**——2026-02 起 Intel 虚拟实例可开嵌套虚拟化，门槛从 ~$2,900/月降到 ~$68/月。

---

## 2. 推荐测试拓扑（按用途分层）

| 用途 | 环境 | 成本 | 说明 |
|---|---|---|---|
| **冒烟/功能验证** | DigitalOcean $6 Droplet 或 GCP Spot | ~$6-20/月 | `deploy/singlenode` 一键装，跑通创建→pause→resume 全链路 |
| **CI 自动化** | GitHub Actions ubuntu-latest | 免费 | 真 microVM 集成测试（见 §4） |
| **性能基准（权威数据）** | Hetzner Auction 裸金属 | €25-40/月 | 无嵌套损耗；M0 基线（恢复延迟矩阵、UFFDIO_COPY 吞吐、ZFS raw file vs zvol 复测）只认裸金属数据 |
| **多节点/温恢复测试** | Hetzner Auction ×2-3 + 私网 | €60-120/月 | 25GbE 方案见《03》§5；预算不足先用 10G |
| **生态兼容验证** | AWS c8i.large / Azure D2s_v5 各按小时开 | 按小时 <$0.1/h | 验证"用户在主流云上能跑" |

**基准数据发布规范**：所有公开 benchmark 注明环境三元组 `(裸金属|嵌套@厂商, CPU 型号, 内核版本)`；嵌套环境数据仅作功能参考，README 性能指标只引用裸金属数据。

## 3. 单机实测流程（对应 deploy/singlenode）

1. 环境检查：`ls /dev/kvm`；`egrep -c '(vmx|svm)' /proc/cpuinfo`（嵌套环境确认 ept/unrestricted_guest 更佳）
2. 依赖安装：Firecracker v1.16.x 二进制、ZFS（数据盘所在 NVMe 建 pool）、官方 microvm guest kernel（6.1）
3. 宿主加固基线（测试环境最小集）：cgroups v2、禁 swap、`unprivileged_bpf_disabled=1`；生产全集见《04》§5
4. 启动 `embervm dev`（单进程模式）→ SDK/CLI 冒烟：create → exec → 写 10GB 数据盘 → pause → resume（计时）→ 归档 → 冷恢复（计时）
5. 跑 `test/bench` 基线套件，产出与《04》§2 延迟预算模型对照的实测报告

## 4. CI 策略（GitHub Actions）

- **单元测试**：无 KVM 依赖，全组件 mock，任意 runner
- **集成测试**（ubuntu-latest，x86_64）：真 Firecracker microVM——需先放开 /dev/kvm 权限：
  ```yaml
  - name: Enable KVM
    run: |
      echo 'KERNEL=="kvm", GROUP="kvm", MODE="0666", OPTIONS+="static_node=kvm"' \
        | sudo tee /etc/udev/rules.d/99-kvm.rules
      sudo udevadm control --reload-rules && sudo udevadm trigger --name-match=kvm
  ```
  覆盖：boot、snapshot/restore、uffd 懒加载、netns 生命周期；矩阵测 Firecracker N 与 N-1 版本
- **每夜基准**：self-hosted runner（Hetzner Auction 裸金属，兼作基准机）跑完整 bench，结果自动发布 gh-pages 看板；公共仓库 self-hosted runner 有安全风险 → 仅跑受保护分支、runner 用一次性 microVM 隔离（actuated 模式，正好吃自己的狗粮）
- **限制**：arm64 CI 无 KVM，arm64 支持仅编译验证，功能测试待自备 arm 裸金属
